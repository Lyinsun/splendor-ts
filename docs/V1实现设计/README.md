# V1 实现设计 README

V1 目标是交付可玩的多人线上 MVP，而不是一次性完成账号、匹配、持久化和完整美术生产管线。

阅读顺序：

1. [01-MVP 实现总览与目录骨架](./01-MVP实现总览与目录骨架.md)
2. [02-Dashboard 前端开发指导](./02-Dashboard前端开发指导.md)

## V1 成功标准

- 两个浏览器窗口可以加入同一房间。
- 服务端可以结算合法行动并拒绝非法行动。
- WebSocket 可以同步权威房间状态。
- Dashboard 能完成创建、加入、开始、拿资源、保留、购买和查看胜负。
- 类型检查、单元测试和 Dashboard build 通过。
