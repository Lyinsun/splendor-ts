# MVP 设计

## 系统定位

本项目是一个多人线上桌游 MVP。规则参考《璀璨宝石》的资源引擎，但主题使用原创元素伙伴，不使用官方宝可梦角色、名称或图像。

核心目标：

- 多个玩家在同一房间中实时同步。
- 服务端是唯一游戏事实写入者。
- 前端只提交意图，不直接改写资源、卡牌、分数或回合。
- MVP 先覆盖可玩闭环，不实现账号系统、公网匹配、AI 托管和持久化数据库。

## 分层

| 层 | 目录 | 职责 |
| --- | --- | --- |
| domain | `src/game/domain` | 卡牌、资源、玩家状态、行动校验与结算 |
| application | `src/game/application` | 房间创建、加入、开始、行动编排与广播事件 |
| infrastructure | `src/game/infrastructure` | WebSocket 房间广播 |
| delivery | `src/gateway/http` | HTTP API、静态资源、请求 DTO 映射 |
| dashboard | `frontend/dashboard` | 浏览器操作台，只通过 API/WS 读写 |

## 规则摘要

- 资源类型：火、水、草、电、念力，另有万能棱晶。
- 回合行动三选一：
  - 拿 3 个不同元素资源。
  - 拿 2 个同元素资源，要求银行该元素至少 4 个。
  - 保留 1 张市场卡，若有棱晶则获得 1 个。
  - 购买 1 张市场或保留卡。
- 每张已购买伙伴卡提供永久元素徽章，后续购买同元素费用降低。
- 达成道馆导师的徽章需求后，回合结束自动邀请 1 位导师并得分。
- 任一玩家达到 15 分后进入终局，当前轮结束后结算胜者。

## 多人同步

- HTTP action 成功后返回权威 `GameState`。
- WebSocket `/ws/rooms/:roomId` 广播 `room_state`。
- 客户端断线后可重新 `GET /v1/rooms/:roomId` 恢复。

## MVP 非目标

- 不使用官方 IP 美术资源。
- 不实现完整 Splendor 规则书中的所有边角规则，例如超过 10 个资源后的弃牌选择。MVP 直接阻止会超过上限的行动。
- 不实现数据库持久化；进程重启后房间清空。
