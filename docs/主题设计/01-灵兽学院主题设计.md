# 灵兽学院主题设计

## 一、定位

`creature-academy` 是面向“宝可梦版璀璨宝石”诉求的原创收集灵兽主题。它保留大众熟悉的抽象体验：属性、伙伴收集、训练成长、徽章挑战和终局王牌，但不使用官方宝可梦名称、角色、造型、Logo、卡面、音效或商标化 UI。

主题只改变展示层：

- `themeId` 决定 Dashboard 文案、hero、卡图和卡牌训练笔记。
- 服务端仍以 `CompanionCard.id`、`element`、`tier`、`cost`、`points` 和 `GameState` 作为权威事实。
- 任何主题资源都不能决定卡牌是否存在、费用、分数、回合或胜者。

## 二、背景抽象

本主题将《璀璨宝石》式资源引擎映射为“学院训练桌”：

| 规则概念 | 灵兽学院展示 |
| --- | --- |
| 宝石资源 | 元素能量 |
| 黄金 token | 万能棱晶 |
| 发展卡 | 元素灵兽伙伴 |
| 永久折扣 | 元素徽章 |
| 贵族 | 道馆导师 |
| 声望分 | 荣耀 |

灵兽学院设定为一个五属性训练学院。玩家不是战斗结算，而是在同一张训练桌上收集能量、邀请伙伴、积累徽章，并通过导师考核触发终局。

参考资料只用于抽象设计边界：

- Space Cowboys 的 Splendor 产品资料用于确认宝石、发展卡、贵族与声望分这些公开规则形状。
- Pokemon 官方站点与 Pokemon TCG 官方站点用于确认属性、收集、训练和进阶是大众可识别的抽象语义。
- Pokemon legal 页面用于提醒项目避免官方商标、角色、美术和受保护表达。

## 三、卡牌内容设计

服务端卡组仍使用 `src/game/domain/content.ts` 中的 45 张 `CompanionCard`。灵兽学院在前端展示层为每张卡补一条训练笔记，文件为 `frontend/dashboard/src/presentation/creatureAcademy.ts`。

训练笔记的设计原则：

- 每张卡都有 `zh-CN` 与 `en-US` 文案。
- 文案强调学院课程、属性路线、徽章成长或终局冲分。
- 文案不描述额外技能，不暗示服务端不存在的规则能力。
- 文案不使用官方宝可梦术语、角色名或世界观名词。

卡组分层：

| 等级 | 展示定位 | 玩法语义 |
| --- | --- | --- |
| Tier 1 | 初阶 / Novice | 建立元素徽章基础，少量一分卡提示路线 |
| Tier 2 | 进阶 / Evolved | 中盘路线成形，开始出现 2-3 分伙伴 |
| Tier 3 | 巅峰 / Apex | 高费用高分王牌，推动 15 分终局 |

属性定位：

| 属性 | 展示关键词 | 路线感 |
| --- | --- | --- |
| fire | 火场、锻炉、日照、高塔 | 爆发、冲分、终局推进 |
| water | 雾池、泡泡课、礁石馆、深潮 | 稳定、转线、资源掌控 |
| grass | 温室、林冠、古树、世界种子 | 成长、折扣积累、后发得分 |
| electric | 雷电课、磁轨、云廊、极光回路 | 节奏、加速、压缩回合 |
| psychic | 梦境课、星盘、记忆回廊、新星殿堂 | 规划、均衡、多系支撑 |

## 四、配图策略

当前 MVP 使用元素/等级图集策略：

```text
assets/splendor-monsters/themes/creature-academy/
  arena-hero.png
  cards/
    card-art-atlas.png
    fire-t1.png
    fire-t2.png
    fire-t3.png
    ...
    psychic-t3.png
```

`card-art-atlas.png` 是 5 列 x 3 行原创图集：

- 列：`fire`、`water`、`grass`、`electric`、`psychic`
- 行：`tier 1`、`tier 2`、`tier 3`
- 裁切产物：`cards/<element>-t<tier>.png`

前端使用 `card.element` 和 `card.tier` 派生图片路径。这样每张卡都有配图，同时控制 MVP 资产数量和加载成本。若后续需要逐卡插画，可新增 `per-card` 策略，但必须同时更新：

- `assets/splendor-monsters/asset-index.json`
- `frontend/dashboard/src/presentation/themes.ts`
- `docs/V1实现设计/02-Dashboard前端开发指导.md`
- 对应 `image-generation/<theme-id>/` manifest

## 五、实现点

- 主题注册：`frontend/dashboard/src/presentation/themes.ts`
- 卡牌训练笔记：`frontend/dashboard/src/presentation/creatureAcademy.ts`
- 卡图渲染：`frontend/dashboard/src/App.tsx`
- 主题样式：`frontend/dashboard/src/styles/dashboard.css`
- 资产索引：`assets/splendor-monsters/asset-index.json`
- 生成记录：`assets/splendor-monsters/image-generation/creature-academy/`

## 六、验收标准

- Dashboard 主题下拉包含 `灵兽学院 / Creature Academy`。
- 切换主题后 hero 使用 `themes/creature-academy/arena-hero.png`。
- 市场和保留区卡牌显示 `creature-academy/cards/<element>-t<tier>.png`。
- 卡牌显示中英双语主题训练笔记。
- 控制台无前端错误。
- `npm run typecheck`、`npm test`、`npm run build:dashboard` 通过。
